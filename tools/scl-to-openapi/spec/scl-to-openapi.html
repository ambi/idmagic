GenerateOpenApi

publicread-onlyidempotentscl-to-openapi/interface/GenerateOpenApi

http binding を持つ interface を OpenAPI 3.1 Operation に変換する。public operation は security を空配列、protected operation は bearer security と x-scl-access を持つ。 requires、ensures、resource、policy と AuthorizationMetadata を vendor extension に保持する。

Scenarios 5

主成功経路と extension で表す black-box の受け入れ例。

public interface は unauthenticated operation になる

Maintainerscl-to-openapi/scenario/public interface は unauthenticated operation になる
Main success
  1. public access の HttpBinding を GenerateOpenApi に渡す
  2. Operation の security は空配列になる

protected interface は security metadata を保持する

Maintainerscl-to-openapi/scenario/protected interface は security metadata を保持する
Main success
  1. policy と resource を持つ protected HttpBinding を GenerateOpenApi に渡す
  2. Operation は bearer security と x-scl-access を持つ

interface contract は記述として保持される

Maintainerscl-to-openapi/scenario/interface contract は記述として保持される
Main success
  1. requires と ensures を持つ HttpBinding を GenerateOpenApi に渡す
  2. Operation の x-scl-requires と x-scl-ensures に原文が保持される

policy 集合から action group が決定的に生成される

Maintainerscl-to-openapi/scenario/policy 集合から action group が決定的に生成される
Main success
  1. 複数の protected interface を BuildAuthorizationMetadata に渡す
  2. policy 集合が同じ action は同一 group に安定順で含まれる

tool spec から解決可能な OpenAPI が生成される

Maintainerscl-to-openapi/scenario/tool spec から解決可能な OpenAPI が生成される
Main success
  1. tool の SCL を GenerateOpenApi に渡す
  2. 全 $ref が components/schemas に解決する
