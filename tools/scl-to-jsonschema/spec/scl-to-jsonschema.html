Glossary 5

曖昧語、別名、翻訳、外部標準語を説明する補助用語集。

SclBundle

context map である SCL 文書 (root) と、そこから参照される各 context 文書を 読み込んでまとめたもの。単一 context 文書だけでも成立する。

aliases —
Model

SCL の models セクションの 1 エントリ。kind は entity / value_object / event / enum / error。

aliases —
ModelDef

1 つの Model から導かれた JSON Schema 断片。出力文書の $defs に Model 名で格納される。

aliases —
GeneratedSchema

bundle 内の全 Model を $defs に集約した単一の JSON Schema 2020-12 文書。 Model 間参照は #/$defs/<ModelName> で解決する。手書きせず常に再生成する。

aliases —
DanglingRef

GeneratedSchema 内の $ref のうち、対応する $defs エントリが存在しない参照。健全な生成では 0 件。

enum モデルは string enum になる

Steps
  1. kind が enum で values を持つ Model を含む SCL を用意する
  2. scl-to-jsonschema で生成する
  3. 対応する $defs エントリが type string と enum 値配列を持つ

entity モデルは required 付きオブジェクトスキーマになる

  2. scl-to-jsonschema で生成する
  3. optional でないフィールドだけが required に入る
