# idp-ADR-086: 管理コンソールおよびマイページの UI 一貫性、ナビゲーション、ページタイトルおよび用語の統一に関する方針

## ステータス

採用。`ui/` 内の各ページ実装に反映。

## コンテキスト

管理コンソール、マイページにおいて「詳細画面から直接編集できる」「編集フォームがモーダルである」「一覧行のアクションボタンが画面によって異なる」「ページタイトルがすべて同じ」といった UI / UX の揺らぎが存在していた。
これらをアドホックに解決すると将来追加される画面で再発するリスクがあるため、一貫した UI ポリシーを策定し、既存画面をそれに従って是正する必要があった。

また、UI 内で「監査ログ」と「監査イベント」という用語が混在しており、バックエンドの SCL 語彙（`AuditEvent`）と乖離していたため、これらを「監査イベント」に統一する。

## 決定

1. **詳細・編集のナビゲーション分離 (Detail-then-Edit Policy)**
   - 設定やリソースの追加・編集を行う際、最初に入力フォームを見せるのではなく、まずは現在の設定やリソース情報を確認できる閲覧（詳細）画面を提示する。
   - 編集はその詳細画面から明示的な操作（編集ボタンの押下等）を経て、専用 of 別画面（別URL）へ遷移して行う。
   - ユーザー登録およびグループの追加・編集、ならびにマイページ（アカウント情報）の編集をモーダルダイアログから専用画面（それぞれ `/admin/users/new`, `/admin/groups/new`, `/admin/groups/:id/edit`, `/account/profile/edit`）に移行する。
2. **一覧行アクションの統一**
   - 一覧画面の各行に用意されるアクションボタン（詳細、編集、削除等）は、三点リーダー（ケバブメニュー）に隠さず、画面上に直接ボタンとして表示してよい。
   - 破壊的操作（削除等）は赤色系（tone='danger'）のボタンで示す。
3. **ページタイトルの動的設定**
   - 各画面の `document.title` に、現在地が分かるタイトル（例: 「ユーザー | IdMagic 管理コンソール」）をルーティングと連動して動的に設定する。
4. **用語の統一**
   - UI 内での「監査ログ」表記を「監査イベント」へ統一し、SCL の `AuditEvent` / `audit_events` と整合させる。

## 却下した代替案

- **モーダルダイアログの維持**: モーダル内に入力要素やエラー表示が凝縮されると、画面幅が制限される中でのスクロールや表示のずれが発生しやすく、URL の変化を伴わないためにブラウザの「戻る」ボタンと整合しないため、却下した。

## 影響

- 新規に画面を追加する際、詳細画面と編集画面を個別の URL (TanStack Router) に分割する。
- 画面に応じた `PAGE_TITLES` エントリを `ui/src/routes/-page.tsx` に追加する。
